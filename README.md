## вңҰ Overview

**GAN Face Editor** is a full-stack web application that exposes the power of [InterFaceGAN](https://github.com/genforce/interfacegan) through a sleek browser UI. Load a pre-trained GAN, sample random face images, then push and pull facial attributes вҖ” age, gender, smile, pose, eyeglasses вҖ” in real-time using intuitive sliders.

```
Browser UI  в”Җв”Җв–ә  FastAPI Backend  в”Җв”Җв–ә  GAN Generator  в”Җв”Җв–ә  Boundary Vectors
    в–І                                                              в”Ӯ
    в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ Base64 PNG в—„в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

> **No Jupyter. No Google Colab.** Just a clean web UI you can run anywhere.

## вңҰ Features

| Feature | Description |
|---|---|
| рҹ§  **Multi-model Support** | Switch between `stylegan_ffhq`, `stylegan_celebahq`, and `pggan_celebahq` |
| рҹ”Җ **W / Z Latent Space** | Toggle between Z-space and disentangled W-space for StyleGAN |
| рҹҺІ **Seeded Sampling** | Reproducible face generation with a numeric seed (0вҖ“1000) |
| рҹ–јпёҸ **Batch Synthesis** | Generate 1вҖ“8 faces side-by-side in a single pass |
| рҹҺӣпёҸ **5 Attribute Sliders** | Age В· Eyeglasses В· Gender В· Pose В· Smile (range: вҲ’3.0 вҶ’ +3.0) |
| вҡЎ **Auto-apply on Release** | Edits fire automatically when you release a slider |
| рҹ“җ **Configurable Viz Size** | Choose output resolution from 64 px to 512 px |
| рҹ“Ӣ **Live Activity Log** | Timestamped log of every API call and result |
| рҹҢҗ **Zero-reload Architecture** | Everything happens over REST вҖ” no page refreshes |

## вңҰ Installation

### 1 В· Clone the repository

```bash
git clone https://github.com/your-username/gan-face-editor.git
cd gan-face-editor
```

### 2 В· Create a virtual environment *(recommended)*

```bash
python -m venv .venv
source .venv/bin/activate        # Linux / macOS
.venv\Scripts\activate           # Windows
```

### 3 В· Install dependencies

```bash
pip install -r requirements.txt
```

## вңҰ API Reference

All endpoints accept and return JSON. Base URL: `http://localhost:8000`

### `GET /status`
Returns current server state.

```json
{
  "model_loaded": true,
  "model_name": "stylegan_ffhq",
  "latent_space_type": "W",
  "has_samples": true
}
```

---

### `GET /models`
Lists all available models and latent space options.

```json
{
  "models": ["pggan_celebahq", "stylegan_celebahq", "stylegan_ffhq"],
  "latent_space_types": ["Z", "W"]
}
```

---

### `POST /load_model`
Loads a GAN model and its boundary vectors into memory.

**Request body:**
```json
{
  "model_name": "stylegan_ffhq",
  "latent_space_type": "W"
}
```

**Response:**
```json
{
  "status": "ok",
  "model": "stylegan_ffhq",
  "latent_space": "W"
}
```

---

### `POST /sample`
Samples random latent codes and synthesizes face images.

**Request body:**
```json
{
  "num_samples": 4,
  "noise_seed": 42
}
```

**Response:**
```json
{
  "status": "ok",
  "image": "<base64-encoded PNG>",
  "num_samples": 4
}
```

---

### `POST /edit`
Applies attribute offsets to the cached latent codes and returns edited faces.

**Request body:**
```json
{
  "age":        1.5,
  "eyeglasses": 0.0,
  "gender":    -1.0,
  "pose":       0.5,
  "smile":      2.0,
  "viz_size":   256
}
```

**Response:**
```json
{
  "status": "ok",
  "image": "<base64-encoded PNG>"
}
```

> All attribute values are clamped to **[вҲ’3.0, +3.0]**. `viz_size` must be between **64** and **512**.

## вңҰ Troubleshooting

**`Model not loaded` error**  
вҶ’ Make sure you've clicked **Load Model** before sampling or editing.

**`No latent codes` error**  
вҶ’ Click **Generate Samples** before applying edits.

**Boundary `.npy` file not found**  
вҶ’ Confirm your `boundaries/` directory contains the correct files for the selected model and latent space type. W-space files end in `_w_boundary.npy`.

**CORS error in browser**  
вҶ’ The FastAPI backend has CORS enabled for all origins by default. If you're behind a reverse proxy, ensure it forwards the `Origin` header.

**Slow inference**  
вҶ’ Enable GPU support by ensuring PyTorch detects CUDA: `python -c "import torch; print(torch.cuda.is_available())"`. The generators automatically use the available device.
